网络隐式保护的再延申

前言

律师: 我有当事人有充分的不在场证明,这是他的浏览器记录,足以证明他在那天晚上没有杀人。

男子：人是我杀的…

在现实生活中，我们需要自己的“个人空间”，其一个重要的原因就是，个人空间提供了一处保护个人隐私的环境。它允许人们在不受打扰的环境中独自思考、放松和进行个人活动，而不必担心被他人窥探或干扰。

而在网络空间里，我们又如何保护我们的隐私呢？

最常见的就是对数据进行加密：

数据加密：

在互联网通信和数据传输中使用加密技术，如 SSL/TLS 协议，以保护数据的机密性。加密可以防止未经授权的访问者在数据传输过程中拦截、窃取敏感信息。

最常见的HTTPS，比如在浏览器中尝试访问百度的网址baidu.com，最终访问的网址是https://baidu.com。

这意味着百度和用户之间的通信使用了安全套接层协议（SSL/TLS）来加密数据传输。这确保了在数据从用户的计算机发送到网站服务器，以及从网站服务器返回到用户的计算机的过程中，数据是经过加密的。

这样，我们在百度搜索、浏览的内容，在数据在互联网传输时就不会轻易被第三方知道。

但是这样，“百度”还是知道谁浏览了什么内容，就像是你去图书馆查资料，会有访客记录。如果我们不希望自己查资料的事情被人知道，该怎么办呢？

现实里，我们可以叫“我的一个朋友”去查，然后他再告诉我，这样图书馆的访客记录就不是自己，只有朋友不开口，不就没人知道自己查资料的事情了吗？

另外，有份资料只有在B地有，但是我们现在在A地，不方便去B地，但是B地有个朋友，叫他查了资料再转述给我们，不是更方便吗？

在网络空间，你可以拥有一个或多个这样的朋友，他们充当“显眼包”，在台前做这些事情，我们安居幕后。

在互联网的这个朋友，我们可以称呼他为代理服务器。

代理服务器

代理服务器是一种位于客户端和目标服务器之间的中间服务器，它充当客户端与目标服务器之间的中转点。

当客户端发送请求时，代理服务器接收请求并将其转发给目标服务器，然后将目标服务器的响应返回给客户端。代理服务器可以在客户端和目标服务器之间执行多种功能，例如代理请求、缓存数据、过滤内容等。

在代理服务器上，我们通过代理工具，使用多种协议来搭建代理服务。

比如可以选择Vmess、Vless、socks协议用于代理服务器和客户端之间进行通信。选择TLS,XTLS来加密数据。

有了代理服务器，我们不仅更方便访问某些数据和服务，我们访问互联网的真实IP地址被隐藏了，网络空间的隐私有了一定的保护，那么是否高枕无忧了呢？

DNS泄露

DNS（Domain Name System）是互联网中用于将将人类可读的域名转换为计算机可理解的 IP 地址的分布式命名系统。

使用代理服务器时，DNS服务有可能会泄露我们的真实IP地址。通常我们可以使用下面的网站来判断是否存在DNS泄露：

IPleak：https://ipleak.net/

browserleaks：https://browserleaks.com/ip

以上两个网站可以显示上网设备发送DNS请求的IP地址。

比如我们的代理服务器位于新加坡，但是在IPleak可以看到位于中国的DNS地址，那么就说明DNS服务泄露了上网设备的真实IP地址。

为什么会发生DNS泄露

首先我们需要知道，DNS请求也是流量，为什么会产生DNS泄露，就是因为DNS流量没有走代理软件，比如以下方式DNS请求产生的流量：

  1. 直接 DNS 查询：

    当你使用代理服务器时，你的计算机或网络配置可能会直接发送 DNS 查询请求，而不通过代理服务器。这样，你的真实 IP 地址将被暴露给 DNS 服务器，从而泄露你的身份。确保你的计算机和网络设置正确地配置了代理服务器，以便所有的 DNS 查询都通过代理进行。

  2. IPv6 泄露：

    如果你的网络支持 IPv6，并且你的代理服务器只能处理 IPv4 流量，那么你的计算机在进行 DNS 查询时可能会绕过代理服务器，直接使用 IPv6 进行查询。这样，你的真实 IP 地址就会通过 IPv6 泄露出去。你可以禁用 IPv6，或者确保代理服务器支持 IPv6 流量，以防止此类泄露。

  3. 特定应用程序的 DNS 查询：

    有些应用程序可能会绕过代理服务器进行 DNS 查询，而直接使用操作系统配置的 DNS 服务器。这种情况下，这些应用程序可能会泄露你的真实 IP 地址。确保你的操作系统和应用程序正确地配置了代理服务器，以确保所有的 DNS 查询都通过代理进行。

避免DNS泄露

我们可以使用Proxifier + Clash/V2ray 来避免DNS泄露。

Proxifier

Proxifier 是一款网络工具，用于在应用程序级别捕获和重定向网络流量。它在操作系统的网络栈之上工作，并允许你将特定应用程序的网络流量路由到指定的代理服务器或其他网络目标。

Proxifier 捕获的流量是在传输层和应用层之间的流量。具体而言，它可以捕获和操作以下类型的流量：

  1. TCP 流量： Proxifier 可以捕获和处理基于 TCP（传输控制协议）的应用程序流量。这包括通过 TCP 连接发送和接收的数据，例如网页浏览、电子邮件、文件传输等。
  2. UDP 流量： Proxifier 也可以捕获和处理基于 UDP（用户数据报协议）的应用程序流量。UDP 是一种无连接的传输协议，常用于实时音视频传输、游戏数据等。
  3. DNS 查询： Proxifier 可以捕获并重定向应用程序发出的 DNS（域名系统）查询请求。这使得你可以将 DNS 查询路由到特定的 DNS 服务器或代理服务器。

Proxifier可以捕获我们设备上网产生的上述流量，这个时候我们将这些流量分流，设置不同的出口，这样所有的DNS请求流量的出口就能被我们控制了。
